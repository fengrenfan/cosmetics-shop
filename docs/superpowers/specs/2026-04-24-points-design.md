# 积分功能设计文档

## 1. 概述

商城积分功能，实现购物返积分和积分抵扣现金。

### 积分规则
- **固定奖励**：每笔订单完成后奖励 10 积分
- **消费返积分**：每消费 10 元奖励 1 积分（20元=2积分，以此类推）
- **积分抵扣**：满 500 积分起可使用，100 积分 = 1 元
- **积分有效期**：按批次管理，每批积分获得后 12 个月过期（FIFO 抵扣）

## 2. 数据库设计

### 2.1 用户表修改 (user)

新增字段：
- `points` INT DEFAULT 0 - 当前可用积分余额

### 2.2 积分记录表 (point_log)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| user_id | INT | 用户ID |
| type | TINYINT | 1=收入, 2=支出, 3=过期 |
| points | INT | 积分数量（正数） |
| balance | INT | 变动后余额 |
| source | VARCHAR(20) | 来源：order=订单, exchange=兑换, expire=过期 |
| order_id | INT | 关联订单ID（订单返积分时） |
| remark | VARCHAR(255) | 备注 |
| expired_at | DATETIME | 过期时间（批次过期时间） |
| created_at | DATETIME | 创建时间 |

### 2.3 订单表修改 (order)

新增字段：
- `points_amount` INT DEFAULT 0 - 使用积分抵扣的积分数量
- `points_money` DECIMAL(10,2) DEFAULT 0 - 使用积分抵扣的金额

## 3. 后端接口

### 3.1 获取用户积分余额
- **GET** `/api/points`
- **Auth**: 需要登录
- **Response**: `{ points: number, pendingPoints: number }`

### 3.2 获取积分明细列表
- **GET** `/api/points/logs?page=1&pageSize=20`
- **Auth**: 需要登录
- **Response**: `{ list: PointLog[], pagination: {...} }`

### 3.3 计算订单可用积分抵扣
- **POST** `/api/points/calculate`
- **Body**: `{ total_amount: number, points?: number }`
- **Response**: `{ canUsePoints: number, canUseMoney: number, maxPoints: number }`

### 3.4 订单完成后返积分
- 内部逻辑，在订单状态变为 `completed` 时触发
- 计算：固定10积分 + floor(实付金额 / 10)
- 生成 point_log 记录，expired_at = now + 12个月

### 3.5 积分抵扣扣减（下单时）
- 内部逻辑，在创建订单时扣减用户积分
- 使用 FIFO：优先扣减即将过期的积分批次
- 生成 point_log 记录（type=2 支出）

### 3.6 积分过期处理（定时任务）
- 每天凌晨执行
- 查找过期的 point_log 批次，标记为已过期
- 注意：只处理状态为"收入"且过期时间 < now 的记录

## 4. 前端页面

### 4.1 我的页面
- 显示当前积分余额（已有占位 `userInfo?.points`）
- 点击进入积分明细页面

### 4.2 积分明细页面
- 展示积分收入/支出记录列表
- 每条记录显示：时间、类型（收入/支出）、积分数量、余额、来源

### 4.3 购物车页面
- 显示用户可用积分数量
- 提示：满500积分可用

### 4.4 结算确认页面
- 显示可用积分数量
- 可输入使用积分数（或滑动选择）
- 实时计算抵扣金额
- 规则：输入积分需为100的倍数，最多抵扣订单金额

## 5. 核心算法

### 5.1 返积分计算
```
固定积分 = 10
消费积分 = floor(实付金额 / 10)
本次返积分数 = 固定积分 + 消费积分
```

### 5.2 抵扣计算
```
最低抵扣 = 500 积分
抵扣比例 = 100积分 = 1元
最大抵扣金额 = floor(用户可用积分 / 100)
最大抵扣金额 = min(最大抵扣金额, 订单金额)
```

### 5.3 FIFO 扣减
```
1. 查询用户所有收入且未扣减的积分批次，按过期时间升序
2. 遍历批次，按顺序扣减至所需积分数
3. 每个批次生成一条支出记录
```
